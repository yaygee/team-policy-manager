# 새 정책 문서 생성

`docs/templates/policy-template.md` 템플릿을 기반으로 새 정책 문서를 생성합니다.

## 지시사항

1. 먼저 `docs/templates/policy-template.md` 파일을 읽어서 현재 템플릿 구조를 확인하세요.
2. 사용자가 입력한 정책 제목: `$ARGUMENTS`
3. 아래 규칙에 따라 `docs/policies/` 디렉토리에 새 정책 마크다운 파일을 생성하세요.

## 파일 생성 규칙

- **파일명**: 정책 제목을 kebab-case로 변환하여 사용 (예: "코드 리뷰 정책" → `code-review-policy.md`). 한글 제목은 적절한 영문으로 변환하세요.
- **저장 경로**: `docs/policies/<파일명>.md`
- **이미 같은 이름의 파일이 존재하면** 사용자에게 알리고 덮어쓰지 마세요.

## 템플릿 자동 채움 규칙

템플릿의 플레이스홀더를 다음과 같이 채워주세요:

- `[정책 제목]` → 사용자가 입력한 정책 제목
- `YYYY-MM-DD` (작성일, 최종 수정일, 변경 이력 날짜) → 오늘 날짜
- `[이름]` / `이름` (작성자, 변경 이력 작성자) → git config user.name 값 조회하여 사용
- **상태** → `초안`으로 설정
- **버전** → `1.0` 유지

나머지 섹션(목적, 적용 범위, 정책 내용, 체크리스트, 관련 정책, 참고 자료 등)의 가이드 텍스트는 그대로 유지하여 사용자가 직접 채울 수 있도록 합니다.

## 완료 후

1. 생성된 파일 경로를 알려주세요.
2. 다음 단계를 안내하세요:
   - 각 섹션의 가이드 텍스트를 실제 내용으로 교체
   - 작성 후 `git add` & `git commit`
   - PR 생성하여 팀 리뷰 진행
