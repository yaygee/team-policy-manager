# 기여 가이드

팀 정책 관리 시스템에 기여해주셔서 감사합니다! 🎉

## 📝 새 정책 추가하기

### 1. 브랜치 생성

```bash
git checkout -b policy/정책명
```

### 2. 정책 작성

템플릿을 복사하여 새 정책을 작성합니다:

```bash
cp docs/templates/policy-template.md docs/policies/your-policy.md
```

### 3. 정책 문서 작성 규칙

- **파일명**: 케밥 케이스 사용 (예: `code-review-policy.md`)
- **제목**: 명확하고 구체적으로
- **날짜**: YYYY-MM-DD 형식
- **링크**: 상대 경로 사용

### 4. 커밋 메시지 규칙

```
feat: 새로운 정책 추가
update: 기존 정책 수정
fix: 정책 오류 수정
docs: 문서 개선
```

### 5. Pull Request

1. 변경사항을 커밋하고 푸시
2. GitHub에서 Pull Request 생성
3. 제목과 설명을 명확하게 작성
4. 리뷰어 지정

## 🔍 리뷰 프로세스

1. **자동 체크**: 링크, 포맷 확인
2. **동료 리뷰**: 최소 1명 이상의 승인 필요
3. **최종 승인**: 정책 책임자 승인
4. **병합**: main 브랜치에 병합

## ✅ 체크리스트

Pull Request 전에 확인하세요:

- [ ] 정책 템플릿 구조를 따름
- [ ] 작성일과 작성자 정보 포함
- [ ] 관련 정책 링크 확인
- [ ] 오타 및 문법 검사 완료
- [ ] 커밋 메시지 규칙 준수

## 💡 팁

- **명확하게**: 모호한 표현 피하기
- **구체적으로**: 예시 포함하기
- **간결하게**: 불필요한 내용 제거
- **일관되게**: 기존 정책과 스타일 맞추기
